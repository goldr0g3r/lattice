feat(search): query DSL + parser

Implements the v0.3 hybrid query grammar from ADR-0018: a hand-rolled
lexer + recursive-descent parser produces a typed Query AST and
surfaces parse errors with byte-range spans so the v0.3 search modal
(#43) can underline the offending region. Covers every operator the
v0.3 epic acceptance calls out (free text, tag: / path: / created:,
quoted phrase, prefix, fuzzy, boolean OR, negation, parens) and ships
~30 unit tests that pin the AST shape.

Pure lattice-search work; the Tantivy execution layer + the
search_query Tauri command + the modal UI come in v0.3 PR E (#43).

- core/lattice-search/src/query.rs — new module with:
  * Query AST (#[non_exhaustive] for v0.9 plugin SDK extensibility):
    All, Term, Phrase, Prefix, Fuzzy{term, distance}, Field{field,
    value}, Date{field, op, date}, And, Or, Not.
  * DateOp enum (Gt, Ge, Lt, Le, Eq) — maps directly to a SQL WHERE
    clause in the v0.3 PR E executor.
  * Lexer: hand-rolled, handles whitespace, identifiers with internal
    hyphens (`bge-small`, `2026-01-01` lex as one token), quoted
    strings with \" / \\ escapes, parens, colon, wildcard, tilde,
    minus, comparison operators (<, <=, >, >=, =), and the `OR`
    keyword (uppercase-only so users can search for the word "or"
    without escaping).
  * Recursive-descent parser with precedence OR < AND < NOT < ATOM;
    flattens nested And/Or so `a b c` becomes one And(vec![a,b,c]);
    handles `--foo` → foo (double-negation collapse via recursion);
    refuses date operators on non-date fields with a clear "use
    created or updated" hint BEFORE attempting to parse the value.
  * Field operators recognised natively: `tag:foo`, `path:Eng/`,
    `created:>2026-01-01`, `updated:<=2026-05-15`, with shorthand
    `created:2026-01-01` for implicit equality.
  * DEFAULT_FIELDS = ["title", "body", "tags"] + TITLE_BOOST = 3.0
    locked here so the PR E executor reproduces the boost without
    drift. DEFAULT_FUZZY_DISTANCE = 1; fuzzy distance caps at 2 to
    match Tantivy.
  * Empty / whitespace-only / "*" all parse to Query::All.
- core/lattice-search/src/error.rs — new InvalidQuery{query, span,
  reason} variant. The span is std::ops::Range<usize> so the modal
  can underline the byte slice with one substring call.
- core/lattice-search/Cargo.toml — adds chrono (was implicit via
  ts-rs only; query module needs NaiveDate directly).
- core/lattice-core/src/error.rs — new LatticeError::InvalidQuery
  {query, reason, span_start, span_end} variant. ts-rs export, From<
  SearchError> impl special-cases InvalidQuery and passes through
  span_start/span_end (split because ts-rs can't represent Range
  natively without a custom type binding). Other SearchError variants
  continue to lift to LatticeError::Search { message } as PR #64
  established.
- core/lattice-core/tests/error_snapshot.rs — adds
  invalid_query_variant_shape locking the JSON shape:
  { "kind": "invalid_query", "details": { "query", "reason",
  "span_start", "span_end" } }.
- 30 new unit tests in query::tests covering: empty matches all, *
  matches all, bareword, implicit AND, three-word AND flatten, OR
  keyword vs lowercase or, precedence (OR below AND), parens
  override, single + double negation, dashed-identifier-is-one-term
  (bge-small), quoted phrase (with escape), prefix, fuzzy default +
  explicit distance + cap at 2, field-scoped term + phrase + prefix,
  date >, <=, implicit =, negated field query, complex combined
  query, every error path (unterminated quote, unclosed paren,
  unbalanced close, bad date, date op on non-date field, empty
  field value), whitespace tolerance, nested parens, and the
  DEFAULT_FIELDS / TITLE_BOOST constants didn't drift from ADR-0018.
- Package totals: lattice-search 45/45 passing (was 11/11),
  lattice-core 80/80 passing (was 75/75 after #64).

Validated locally — cargo fmt --all --check, cargo clippy --workspace
--all-targets --all-features -D warnings, cargo test (lattice-search,
lattice-core), all clean.
